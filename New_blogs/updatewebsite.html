1.批量修改网站页面
经验：个人用的是editplus，小巧方便，利用它批量打开新文件，然后下方切换页面，修改后可一键保存。
（1）首先把需要的页面批量复制下载下来，然后用editplus批量打开，下方有标签页，改完一个切换到下一个，都改完后点一键保存，保存所有文件。
（2）首先观察windows文件，都是一个网页跟着一个bak，如果有的没跟着，说明没改，确保所有的都有bak文件都改过了，利用windows的按照类型排序，把bak文件（eidtplus自带的备份文件，我一般不删，都留着）和网页分开，按住shift批量选择网页，粘贴上传到网站服务器上，替换旧的网页。

3.ftp报错第一种（FTP文件夹错误）
将文件复制到FTP服务器时发生错误。请检查是否有权限将文件放到该服务器上。
详细信息:
200 Type set to I.
227Entering Passive Mode (122,114,49,4,4,87).
125 Data connection already open;Transfer starting.
426 Connection closed;transfer aborted.
经验：（1）打开“Internet选项”，选择“高级”选项卡，在设置中把“使用被动FTP（用于防火墙和DSL调制解调器的兼容）”前面的勾去掉，并点击“确定”退出“Internet属性”。
(2)第一个方法没效果，测试了下，删除旧文件，腾出空间来，再复制上新的就行了，可能是空间不足引起的。

4.ftp报错第二种（FTP文件夹错误）
将文件复制到FTP服务器时发生错误。请检查是否有权限将文件放到该服务器上。
详细信息:
与服务器的连接被重置。
经验：（1）.用重新登录下试试，刷新下页面。
（2）刷新下页面。
（3）如果是批量更新，逐个或逐几个文件复制上。
（4）删除旧的不需要的文件，再复制上试试。
（5）删查看文件名中是否有中文。
PS：连接被重置比较常出现，有时候也是过段时间就好了。有时候复制多次，尝试多次，会有其他提示出现。
5.ftp报错第三种（FTP文件夹错误）
将文件复制到FTP服务器时发生错误。请检查是否有权限将文件放到该服务器上。
详细信息:
200 Type set to I.
277 Entering Passive Mode(123,57,298,27,156,66).
451 No mapping for the Unicode character exisits in the target multi-byte code page.
经验：发现是压缩包的名字里有中文，去掉中文就可以了。
