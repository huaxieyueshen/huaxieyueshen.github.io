【LODOP】提示“网页还没下载完毕，请稍等一下再操作.”（文案，初）

提示“网页还没下载完毕，请稍等一下再操作.”
这个是发行包里lodopFuncs.js里的提示，
（图）

原来的c-lodop版本也会有提示“C-Lodop没准备好”

类似的提示还有，提示"WebSocket没准备好，点确定继续"，这个提示是在CLodopfuncs.js里
例如http网站，且是8000端口时，http://localhost:8000/CLodopfuncs.js中
（图）

解决方法：
参考http://www.c-lodop.com/faq/pp3.html
新版修改了该提示，是同一问题，参考链接里的方法。

情况1：点击按钮或其他方式等触发
如果是页面点击后打印，把获取LODOP对象放在打印按钮里触发。参考样例2
http://www.c-lodop.com/demolist/PrintSample2.html
进入页面稍等一会再点击打印按钮


情况2：一进入页面就调用，不是进入点击按钮等触发方式（c-lodop已安装启动）

参考常见问答的三种方法：参考http://www.c-lodop.com/faq/pp3.html

方法一：等待C-Lodop准备好再执行
调用打印方法打印，详见样例22，因样例是混合部署，所以写了判断，c-lodop和lodop两种。
样例22是一进入页面就弹出打印预览（c-lodop以弹出的形式，lodop插件会是内嵌浏览器方式），
没有点击打印按钮，一进入就调用，代码里加了等c-lodop准备好的方法

if (needCLodop()) {
//如果是只运行C-Lodop,只需下边的代码，
//C-Lodop----begin----
window.On_CLodop_Opened=function(){
OpenPreview(); //OpenPreview()是调用打印方法
window.On_CLodop_Opened=null;
};
//C-Lodop----end----
} else
window.onload = function(){OpenPreview();}; //使用lodop控件时的调用

方法二：设置时间延迟执行
setTimeout("print()","500"); //500毫秒后执行print()函数，只执行一次。
function print(){
LODOP.PRINT_INIT("测试打印");
LODOP.ADD_PRINT_TEXT(10,10,300,200,"这是纯文本行");
LODOP.PRINT();
}

方法三：提前自行判断一下之后再打印，自行判断方法如下：
if (LODOP.webskt && LODOP.webskt.readyState==1){ 准备好} else {没准备好}

“网页还没下载完毕，请稍等一下再操作.”或提示“C-Lodop没准备好”
还有一起团其他原因，可参见问答介绍http://www.c-lodop.com/faq/pp3.html
（图）

 

其他现象：总是提示“网页还没下载完毕，请稍等一下再操作.”，刷新页面，进入页面等待也没有效果。
可能原因：c-lodop没安装启动。
查看下c-lodop是否处于启动状态 ，端口是否正常，或http的CORS阻挡，https的证书等，
此问题可参见另一个提示未安装启动的视频，就不多做介绍了：
（图）

c-lodop没安装，想测试lodopFuncs.js自带的提示下载安装，没有提示下载安装，提示这个“网页还没下载完毕，请稍等一下再操作.”

进入页面稍等一会再调用LODOP对象，最新版lodopFuncs.js里，等四个报错都完成，判断的JS都完成了，再调用LODOP对象会有提示
（图）

如果是一进入页面就调用，加延迟调用LODOP对象，等clodopFucns.js里加载完成，报错也完成，会有提示
